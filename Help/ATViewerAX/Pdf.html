PDF support

To support PDF files loading, you must use plugin sLister.dll: file is included in PdfDll folder. This plugin uses "Sumatra PDF" program to show PDF ("Sumatra PDF" program is also included in PdfDll).

These are steps to support PDF:

  • Create the app without plugins usage
  • Correctly set ParentWnd property
  • Copy PdfDll folder to app folder
  • In app initialization add code like this (sample Delphi code): 

      S:= ExtractFileDir(Application.ExeName) + '\PdfDll\slister.dll';
  if not FileExists(S) then
    begin ShowMessage('Not found PDF dll:'#13+ S); Exit end;
  ATViewer1.AddPlugin(S, 'ext:pdf');
  • If you rename PdfDll folder to a different name, you should adjust path to this folder in code too.

If you want you may not use sLister plugin, but then Acrobat/Foxit Reader will be needed on end user's system.
W/o plugin pdf is loaded in "Web" mode (ATViewer1.Mode = vmodeWeb). With plugin pdf is loaded in "Plugins" mode (ATViewer1.Mode = vmodeWLX).
